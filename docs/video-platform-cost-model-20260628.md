# 智选视频平台存储、转码、分发成本模型 - 2026-06-28

## 1. 文档目标

本文件用于定义 `智选` 终局视频平台的成本模型，解决以下问题：

- 在亿万用户级规模下，什么成本最危险
- 当前系统为什么不够省
- 终局系统应该把钱花在哪里
- 哪些地方必须省，哪些地方不能省

本文不追求精确财务数字，而追求正确的成本结构与决策模型。

## 2. 核心原则

### 2.1 平台级成本判断原则

在海量视频业务中，最危险的往往不是单条视频存储，而是以下 4 类成本叠加：

- 热存储成本
- 转码计算成本
- 回源与分发成本
- 低缓存命中导致的重复传输成本

### 2.2 错误省钱方式

以下做法看似省钱，长期反而更贵：

- 让前台整文件直拉导致回源巨大
- 所有视频固定高码率
- 双存多套公开主资产
- 低命中率导致重复回源
- 缺少生命周期治理导致热存堆积

### 2.3 正确省钱方式

真正有效的降本手段是：

- 主分发资产唯一化
- 冷热分层
- 内容自适应码率
- 高缓存命中率
- 热点视频边缘化
- 低价值内容自动降温
- 生命周期回收

## 3. 当前系统的成本结构

### 3.1 当前主链路

当前系统近似为：

1. 原片上传
2. 端侧转码为 MP4
3. 生成 HLS
4. 上传 MP4
5. 上传 HLS
6. 上传封面
7. 前台优先 HLS，失败回 MP4

### 3.2 当前系统的成本问题

当前系统最大的问题不是“功能不可用”，而是“重复主资产过多”。

同一条视频在公开播放链上同时存在：

- 完整 MP4
- HLS 清单与分片
- 封面

这意味着平台在规模放大后会承担：

- 更高热存成本
- 更高对象数管理成本
- 更高生命周期治理复杂度
- 更高对象一致性维护成本

### 3.3 当前系统为什么仍然合理

在平台早期，双存有它的价值：

- 上线速度快
- 回退简单
- 调试方便
- 兼容性更高

但对于亿万用户级目标，它不应作为终局。

## 4. 终局成本结构

### 4.1 两层存储模型

终局平台只保留两层核心存储：

#### 冷层

用途：

- 原片
- mezzanine 母版
- 审计或申诉保留素材

特点：

- 访问频次低
- 生命周期可控
- 以低成本长期保存为目标

#### 热层

用途：

- 唯一线上主分发资产
- manifest
- init segment
- media segment
- 封面

特点：

- 面向高频播放
- 以边缘分发效率和命中率为核心

### 4.2 主分发资产唯一化

终局平台公开播放链只保留一套主资产：

- `HLS + CMAF/fMP4`

不再长期保留：

- 完整 MP4 公开播放版本

### 4.3 为什么这才省

这样做可以直接减少：

- 重复热存
- 重复对象生命周期管理
- 重复数据一致性维护
- 兼容兜底造成的无效常驻存储

## 5. 单视频成本拆解模型

设：

- `S_cold` = 冷层单视频占用
- `S_hot` = 热层单视频占用
- `C_transcode` = 单视频转码成本
- `C_dist` = 单视频分发成本
- `C_meta` = 元数据与审计成本

则平台单视频总成本为：

`C_total = S_cold + S_hot + C_transcode + C_dist + C_meta`

### 5.1 冷层成本

冷层成本的关键不是“绝对字节最小”，而是：

- 是否保留原片
- 是否保留 mezzanine
- 保留多久
- 哪些视频可以在生命周期阶段转入更低成本层

### 5.2 热层成本

热层成本受以下因素影响最大：

- 分发主资产大小
- 视频热度
- 热存保留时长
- 边缘缓存命中率
- 热门内容回源率

### 5.3 转码成本

转码成本受以下因素影响最大：

- 编码格式
- 分辨率
- 帧率
- 并发量
- 是否多档输出

平台若坚持“单一高清主资产”，转码复杂度会低于多档 ABR，但分发压力更高，因此必须用编码效率和缓存命中率对冲。

### 5.4 分发成本

分发成本受以下因素影响最大：

- 用户观看时长
- 完播率
- 回看率
- 热点视频复看频率
- 分片大小
- 缓存命中率

对超大规模平台而言，分发成本通常比单纯存储更危险。

## 6. 成本优化的真正杠杆

### 6.1 不是降分辨率

平台不应通过把推荐流降到标清来省钱。

原因：

- 会破坏平台品牌感
- 会损害推荐流沉浸感
- 会直接降低用户主观体验

### 6.2 真正的杠杆一：内容自适应码率

在固定 `1080 x 1920` 的前提下，根据内容复杂度调码率。

收益：

- 静态内容显著降本
- 高复杂内容仍能保住质量
- 平均存储与平均带宽更可控

### 6.3 真正的杠杆二：高效编码

长期看，平台不能只靠低压缩效率编码。

正确方向：

- 高压缩效率编码作为主方向
- 兼容编码作为受控兜底

收益：

- 同等主观质量下显著降低字节体积
- 直接降低热存与分发成本

### 6.4 真正的杠杆三：主分发资产唯一化

收益：

- 热存直接下降
- 运维复杂度下降
- 迁移与回收成本下降

### 6.5 真正的杠杆四：缓存命中率

对亿万用户级平台来说，缓存命中率就是成本开关。

命中率高意味着：

- 回源少
- 分发成本更可控
- 首帧更快
- 热门视频整体体验更稳

### 6.6 真正的杠杆五：生命周期治理

平台不能让所有视频永久留在热层。

必须按热度和时间推进：

- 热层
- 温层
- 冷层
- 删除或归档

## 7. 推荐流的成本策略

### 7.1 推荐流不能省的地方

- 统一画布质量
- 首帧体验
- 切换速度
- 分片缓存质量
- 热门内容边缘升温

### 7.2 推荐流必须省的地方

- 重复主资产
- 无意义的全量高码率
- 无意义的全量 60FPS
- 无生命周期管理的热存长期堆积

## 8. 横屏流的成本策略

横屏流应单独建模，不与推荐流混用。

原因：

- 横屏流消费习惯不同
- 热点分布不同
- 观看时长结构可能不同
- 存储和缓存命中模型不同

横屏流单独建模有利于：

- 单独核算成本
- 单独治理
- 单独调优

## 9. 亿万用户级平台的关键成本指标

平台上线后必须按天、按频道、按地区、按设备持续观察：

- 单视频平均热存大小
- 单视频平均冷存大小
- 每千次播放平均字节消耗
- 首帧平均时间
- 分片失败率
- 回源率
- 缓存命中率
- 观看 10 秒的平均流量消耗
- 热点视频边缘命中率
- 编码成功率与重试率
- 生命周期迁移成功率

## 10. 终局成本结论

如果 `智选` 要成为亿万用户级国民视频平台，必须明确以下结论：

- 真正的成本敌人不是“高清”
- 真正的成本敌人是“重复主资产 + 低缓存命中 + 无治理热存”
- 推荐流必须保住画质
- 成本必须通过编码效率、主资产唯一化、缓存命中率和生命周期治理来解决

## 11. 决策结论

平台终局应立下以下硬规则：

- 推荐流不降空间分辨率
- 公开播放主链不长期保留完整 MP4
- 主分发资产唯一化
- 冷热分层
- 热点缓存升温
- 生命周期治理强制化
- 成本观测必须进入平台核心指标

这不是“节约一点对象存储”，而是“从架构层避免未来成本失控”。
